# Nudge 技术审查记录

审查日期：2026-06-23  
上游基线：1da2890965b73299a2b0344231c4945ac74f6d0f  
验证环境：macOS 15.7.8、Xcode 26.3、Swift 6.2.4、XcodeGen 2.45.4

## 本轮结论

- 无剩余构建或单元测试阻塞项。无签名 Debug 构建成功，XCTest 共 34 项全部通过。
- 原测试目标错误继承生产 App 的 Info.plist，导致测试 Bundle 的 principal class 也是 NSApplication。XCTest 加载测试时触发 “Creating more than one Application” 并等待至超时。本轮已改为独立生成测试 Bundle plist，并在测试宿主启动时跳过生产副作用。
- 拖拽吸附此前只在前几个事件中尝试判断窗口是否移动，随后仍可能把截图选区或桌面文件拖动识别为窗口拖动，与上游 [Issue #1](https://github.com/mikusnuz/nudge/issues/1) 一致。本轮已改为标题栏命中且观察到窗口真实位移后才激活。
- 快捷键录制框的“✕”此前只清空当前文字，没有更新偏好或热键注册；取消录制后也可能残留 “Type shortcut...”。本轮已加入可持久化的禁用状态并修复取消、冲突和重载顺序。
- AppIcon 的像素槽位原本齐全，但主体占满 1024×1024 画布。本轮按传统 macOS 824×824 keyline 居中重生成全部尺寸。

## 已补测试

- 拖拽状态：非标题栏拖动整次忽略、无窗口位移保持等待、真实窗口位移激活、手势结束后可复用。
- 快捷键偏好：默认、自定义旧数据、禁用、重置、冲突排除当前动作和已禁用动作。
- 快捷键录制视图：取消后恢复原文字，清除后通知持有者并显示未分配。
- 吸附几何：改用固定 CGRect，覆盖奇数尺寸、中央三分之二和各类空结果，避免无屏幕时静默通过。

## 后续重构机会

### P1：可靠性与兼容性

- WindowManager 多处将 AX 返回值强制转换为 AXUIElement 或 AXValue，并忽略 AXValueGetValue、设置位置和尺寸的错误码。异常应用或变化后的系统属性可能造成崩溃或静默失败。建议集中封装类型检查、错误映射和日志。
- SkyLight fallback 依赖私有框架和运行时符号，只能可靠移动窗口；当应用不暴露 AX 窗口时通常也无法完成尺寸调整。应明确它是尽力而为的兼容路径，并评估用受支持 API 替代或对不完整结果给出反馈。

### P2：窗口与显示模型

- getWindowID 匹配失败时退化为 “<pid>-unknown”，同一进程的多个窗口会共享恢复记录；previousFrames 也没有窗口销毁或进程退出后的清理。建议引入稳定窗口身份和生命周期清理。
- 坐标转换散落在 DragSnapManager、WindowManager 和 DisplayHelper，并隐含主屏高度与水平排列假设。建议提取统一坐标转换器，并为上下排列、负坐标和不同缩放比例的多显示器补测试。
- WindowManager 同时承担窗口发现、AX 读写、恢复历史、跨屏循环和私有 API fallback。可按上述边界拆分，但应在增加集成测试后进行。

### P2：隐私与可观测性

- 启动时会向硬编码分析服务发送事件，目前没有偏好开关、隐私说明或失败可观测性。建议确认产品隐私策略，并决定移除、改为明确选择加入，或至少记录用途与数据字段。
- Debug 文件日志固定写入用户主目录，缺少轮转。若继续用于支持场景，建议迁移到统一日志并限制保留量。

### P3：工程维护

- 当前没有 CI。建议至少在 macOS runner 上执行 xcodegen generate、无签名构建和 XCTest，并检查生成后的工程无 diff。
- README.md 宣称 18 个动作，但代码与测试包含 19 个，中央三分之二动作未列入快捷键表。
- Nudge.xcodeproj 是 project.yml 的生成产物且同时提交；后续应在 CI 中防止两者漂移。
- Xcode 26.3 会输出 “Supported platforms for the buildables in the current scheme is empty” 警告，当前不影响构建或测试。升级 XcodeGen/工程格式时应重新确认并清理。
